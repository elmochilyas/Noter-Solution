## Issues found

### BUG-001 ŌĆö (P1) `app/Domain/Services/BookingService.php:100`
- **Status: fixed**
- **What:** `reschedule()` calls `cancel()` which dispatches `BookingCancelled`, but `BookingRescheduled` is never dispatched.
- **Reproduction:** Reschedule any confirmed booking ŌĆö no reschedule notifications fire, reminders aren't re-scheduled.
- **Root cause:** Missing `BookingRescheduled::dispatch()` after the new booking is created.
- **Fix:** Added `BookingRescheduled::dispatch($booking, $newBooking)` after `createPending()` returns.
- **Test added:** `it('reschedule dispatches BookingRescheduled event')` in `BookingServiceTest.php`

### BUG-002 ŌĆö (P1) `app/Domain/Services/BookingService.php:27`
- **Status: fixed**
- **What:** `createPending()` had no `DB::transaction` and didn't call `assertSlotIsFree()`, allowing concurrent requests to double-book the same slot.
- **Fix:** Wrapped creation in `DB::transaction` and added `assertSlotIsFree()` call before persisting.
- **Test added:** `it('createPending throws SlotNotAvailable when slot is already booked')` in `BookingServiceTest.php`

### BUG-003 ŌĆö (P1) `app/Filament/Resources/BookingResource.php:181`
- **Status: fixed**
- **What:** `TimeSlot::fromIso()` called but method didn't exist. Crashed on admin reschedule confirm.
- **Fix:** Added `TimeSlot::fromIso(string $iso, int $durationMinutes): self` static factory method.
- **Test added:** `it('fromIso creates TimeSlot from ISO string and duration')` in `TimeSlotTest.php`

### BUG-004 ŌĆö (P1) `app/Filament/Resources/BookingResource.php:170-177`
