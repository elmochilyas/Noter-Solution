## Bugs found

### B-001 — (P1) `app/Http/Controllers/Webhook/StripeWebhookController.php:65-68`
- **Status: fixed**
- **What:** `new GatewayWebhookEvent(type: ..., data: ...)` missing the required `$id` parameter. `ArgumentCountError` thrown on every webhook.
- **Fix:** Added `id: $event->id`.
- **Tests added:** `it('confirmFromWebhook handles canceled intent as failed')` in `PaymentServiceTest.php` and `PaymentWebhookTest.php`

### B-002 — (P2) `app/Domain/Services/PaymentService.php:53-72`
- **Status: fixed**
- **What:** `payment_intent.canceled` listed as handled type in controller but `confirmFromWebhook()` had no matching branch. Payments stayed `pending` indefinitely.
- **Fix:** Added `elseif ($event->type === 'payment_intent.canceled')` that marks payment as `FAILED` and dispatches `PaymentFailed`.

### B-005 — (P3) `app/Http/Controllers/Webhook/TwilioWebhookController.php` + `ResendWebhookController.php`
- **Status: deferred**
- **What:** No webhook signature verification. Any HTTP POST to these endpoints is accepted.
- **Note:** Stripe webhook has signature verification already.

### B-006 — (P3) `app/Listeners/GenerateCreditNote.php:36`
- **Status: deferred**
- **What:** `nextSequenceNumber()` calls `SELECT nextval('credit_notes_number_seq')` but no migration creates this sequence. Falls back to `max('id') + 1` silently.
